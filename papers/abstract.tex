System prompts for Large Language Model (LLM)-based
coding agents are software artifacts that govern agent behavior, yet
lack the testing infrastructure applied to conventional software. We
present Arbiter, a framework combining formal evaluation rules with
multi-model LLM scouring to detect interference patterns in system
prompts. Applying this framework to the three major coding agents whose
system prompts are publicly available --- Claude Code (Anthropic), Codex
CLI (OpenAI), and Gemini CLI (Google) --- we identified 152 findings
across the undirected scouring phase and 21 hand-labeled interference
patterns in directed analysis of one vendor. We show that prompt
architecture (monolithic, flat, modular) predicts the \emph{class} of
failure but not its \emph{severity}, and that multi-model evaluation
discovers categorically different vulnerability classes than
single-model analysis. One scourer finding --- structural data loss in
Gemini CLI's memory system --- is consistent with an independently filed
vendor bug report and patch that addressed the symptom without resolving
the schema-level root cause. Total cost of cross-vendor analysis:
\$0.27.
